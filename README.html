App Engine Calendar API Sample Instructions

A starter application has been generated for you.

  1. Install Python 2.
  2. Install the Google App Engine SDK for Python.
  3. Download the starter application and unzip it. It will unzip into a directory called calendar-appengine-sample.
  4. Download the client secrets file and overwrite the client_secrets.json file that was included in the application. The client_secrets.json file contains your applications client id and client secret and should be stored securely.

  5. You can run your starter application using the Google App Engine Development Server which is included in the Google App Engine SDK for Python. You can launch the application using the Google App Engine Launcher or, if you're not using Google App Engine Launcher, start the web server with the following command, giving it the path to the calendar-appengine-sample directory: 

    google_appengine/dev_appserver.py calendar-appengine-sample
  6. Visit http://localhost:8080 to see your application running.

Congratulations, you are up and running! At this point you will want to add calls into the Calendar API to the main.py file. Please read the main.py file carefully, it contains detailed information in the comments. For more information on the Calendar API Python library surface you can visit:

https://google-api-client-libraries.appspot.com/documentation/calendar/v3/python/latest/

The Calendar API has been turned on in the Cloud Console. You can see and adjust the OAuth 2.0 project settings by visiting:

https://cloud.google.com/console#/project/116894289487/apiui

Also check out the Python Client Library documentation, and get more information on the Calendar API at:

https://developers.google.com/google-apps/calendar/firstapp
